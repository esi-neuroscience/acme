# ACME (Asynchronous Computing Made ESI) - Agent Instructions

## Package Structure

Single-package Python library for parallel computing via Dask and SLURM.

**Main entrypoints:**
- `ParallelMap` - user-facing context manager in `acme/frontend.py`
- `ACMEdaemon` - computational backend in `acme/backend.py`
- Cluster setup helpers in `acme/dask_helpers.py`:
  - `esi_cluster_setup` - ESI HPC cluster (Frankfurt)
  - `bic_cluster_setup` - CoBIC HPC cluster (Frankfurt)
  - `slurm_cluster_setup` - generic SLURM clusters
  - `local_cluster_setup` - local multiprocessing

**Core modules:**
- `frontend.py` - user interface
- `backend.py` - execution scaffolding
- `shared.py` - utilities, HPC node detection
- `dask_helpers.py` - Dask/SLURM integration
- `logger.py` - logging setup

## Developer Commands

**Python setup**
```bash
source /opt/conda/etc/profile.d/conda.sh
conda activate acme
```

**Installation (editable mode):**
```bash
pip install -e .[dev]
```

**Run all tests:**
```bash
cd acme/tests
./run_tests.sh pytest
```

**Run specific test:**
```bash
cd acme/tests
./run_tests.sh pytest test_pmap.py -k 'test_simple_filter'
```

**Run tox (tests across Python versions):**
```bash
cd acme/tests
./run_tests.sh tox
```

**Build documentation:**
```bash
cd doc
make html
```

**Linting (as run in CI):**
```bash
pip install flake8
flake8 . --count --select=E9,F63,F7,F82 --show-source --statistics  # syntax errors only
flake8 . --count --exit-zero --max-complexity=10 --max-line-length=127 --statistics  # all warnings
```

**Testing locations:**
- Tests in `acme/tests/`
- `pytest.ini` configuration in `acme/tests/pytest.ini`
- `conftest.py` provides cluster-specific fixtures and skips

## Testing Quirks

**Test environment detection:**
- `conftest.py` automatically detects if running on ESI, CoBIC, or other SLURM clusters
- Tests are conditionally skipped based on cluster availability via `is_slurm_node()`, `is_esi_node()`, `is_bic_node()`

**Test runner script:**
- `acme/tests/run_tests.sh` handles both SLURM and local execution
- Automatically wraps pytest in `srun` when on cluster nodes with appropriate partition selection
- Adds coverage options (`--cov=../../acme`)

**Important test prerequisites:**
- Requires SLURM for cluster-specific tests
- Some tests require specific partitions (8GBL, 8GBXS, E880, etc.)
- Test filtering with `-k` flag is common

## Important Conventions

**HPC cluster awareness:**
- Code detects ESI vs CoBIC vs generic SLURM clusters
- Partition and memory defaults differ per cluster
- Functions may return different types (None, Client, SLURMCluster, LocalCluster)

**Space-inefficient result handling:**
- Default behavior: saves results to HDF5 files, one per function call
- Use `result_shape` parameter to stack results in single dataset
- Use `write_worker_results=False` for in-memory collection (not recommended)

**Debugging mode:**
- `ParallelMap` supports `debug=True` parameter to run everything locally
- Enables pdb and IPython debugging

**Logging:**
- Module-wide logging configured in `__init__.py` via `prepare_log("ACME")`
- dask-jobqueue FutureWarnings are suppressed globally

**Only work locally and respect authorship** 
- Use a dedicated branch and never push to remote
- Author your commits as SAIA GWDG Chat AI (do not edit `.git/config` for this)
